Prio-logikk. P1 = blokkerer ende-til-ende-flyt eller mangler GDPR-/forretningskrav. P2 = løfter UX vesentlig. P3 = polish som kan komme i fase 3.

Mobil-strategi. Vi leverer mobil-paritet i to bolker (A3 coach + A2 spiller) i stedet for å sprinkle 21 separate mobil-varianter. Hver bolk inneholder 5 skjermer i én HTML — speiler hvordan komponentene faktisk brukes på telefon.

Tale. A4 leveres som primitiv som så drop-inn i alle relevante komponenter via "diktér"-suffix. Ingen separate tale-states per komponent.

States. Loading/empty/error-arbeid for de 11 "DELVIS"-radene gjøres når komponenten anyway røres for mobil eller compliance. Ingen egen pass — for tett arbeidsmønster.

Snip-strategi. Hver leveranse er én HTML i preview/ · samme mønster som fase 1 · samme tokens · ingen dependency-graf utenfor _shared.css.
