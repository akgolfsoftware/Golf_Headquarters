# Mini-batch 6-B - PlayerHQ Settings + felles layout

**5 skjermer i denne mini-batchen.** Felles moenster: form-baserte settings flater for spilleren (Markus), pluss det felles innstillings-layoutet som binder hele plattformen sammen.

**Generer alle 5 skjermer i denne sesjonen.** Foelg arketype-F-felles-spec (sub-nav, read-only-default, per-felt-lagring, save-bar, farezone).

---

## Felles arketype-F spec

Samme som 6-A. Read-only er default, edit er bevisst handling. Toggles har label + description. Farezone nederst. Save-bar kun naar form er dirty.

PlayerHQ har annen sidebar-stil enn CoachHQ:
- Desktop: enkel venstre-sidebar (240px) med lyst tema
- Mobil: bunn-tab-bar med 4-5 ikoner

## Pakke 5/5: Shared Innstillings-layout (master template)

# AK Golf Platform -- Shared -- Innstillings-layout

## Identitet

- **Produkt:** Shared / cross-cutting
- **URL:** Referanse-skjerm (ikke faktisk URL)
- **Arketype:** F (master-template)
- **HTML-referanse:** `wireframe/screen-deck/shared/cross-cutting/innstillings-layout.html`

## Spec

Innstillings-layout er **mal-skjermen** alle settings-flater i hele plattformen bygger paa. Naar en designer eller utvikler trenger aa vite "hvordan ser en settings-side ut?", er svaret denne skjermen.

## Layout

3 stadier av samme layout vises side-om-side (eller stacked paa mobil):

### Visning A: Empty state
3-kolonne med skeleton-form (4 felt-skeleton-rader).

### Visning B: Read-only default
Sub-nav viser "Profil" som aktiv (accent-bg + venstre 4px accent-border). Form har 4 felt-rader: "Fullt navn: Anders Kristiansen [Endre ->]", E-post, Mobil, Avatar. Save-bar skjult.

### Visning C: Active edit
"E-post"-feltet er i edit-mode: input med fokus, [Avbryt] [Lagre] inline. Save-bar synlig nederst: "1 endring ulagret". Toast i hjoernet: "E-post oppdatert kl 14:32".

### Spec-card (hoeyre side)
Liten dokumentasjons-card viser:
- Sub-nav active: bg-primary/10 border-l-2 border-primary text-primary font-medium
- Sub-nav default: text-muted-foreground hover:bg-muted
- Felt-row: border-b border-border py-4 flex items-center justify-between
- Edit-link: text-sm text-primary hover:underline
- Save-bar: sticky bottom-0 bg-background/95 backdrop-blur border-t
- Farezone: border border-destructive/30 bg-destructive/5 rounded-lg p-6 mt-12

## Pyramide / hierarki regler

1. Sub-nav sticky til venstre (eller dropdown oeverst paa mobil)
2. Form-seksjoner har en H3-tittel + valgfritt description
3. Felter er minst 56px hoeye (touch-target)
4. Read-only foerst -- alltid
5. Save-bar kun hvis multi-felt-form
6. Farezone alltid nederst, separert med 48px+ margin

## OEnsket output

1. Visning A (empty/skeleton) lyst tema
2. Visning B (read-only) lyst tema
3. Visning C (edit + save-bar) lyst tema
4. Visning B moerkt tema
5. Mobil <=640px (sub-nav som dropdown, save-bar full bredde)
6. Spec-card med token-dokumentasjon synlig
