# Mini-batch 6-A - CoachHQ Settings - profil og kontoinnstillinger

**5 skjermer i denne mini-batchen.** Felles moenster: form-baserte settings-flater. Anders bruker disse sjelden men de maa vaere uten friksjon naar han trenger dem.

**Generer alle 5 skjermer i denne sesjonen.** Foelg arketype-F-felles-spec for fellestrekk (sub-nav, read-only-default, per-felt-lagring, save-bar, farezone).

---

## Felles arketype-F spec (gjelder alle skjermer her)

### Layout

```
+--------------------------------------------------+
|  Sidebar   |  Hero (italic editorial)            |
|            +-------------------------------------+
|  CoachHQ   |  Settings-sub-nav (sticky vertikal) |
|  to-lags   |  - Profil (active)                  |
|  rail+nav  |  - Sikkerhet                        |
|            |  - Varsler                          |
|            |  - API + integrasjoner              |
|            |  - Abonnement                       |
|            |                                     |
|            |  ----------+------------------------|
|            |            | Form-seksjon           |
|            |            | +---------------------+|
|            |            | | Felt 1   [Endre ->]||
|            |            | +---------------------+|
|            |            | | Felt 2 (toggle) () ||
|            |            | +---------------------+|
|            |            |                        |
|            |            | [Avbryt] [Lagre]       |
|            +-------------------------------------+
```

### Mostre paa tvers (alle 5 skjermer)

| Element | States aa designe |
|---|---|
| Sub-nav-item | default, hover, active (4px venstre-border + bg accent/10) |
| Felt-row | default, hover (subtil bg-muted/30) |
| "Endre ->" link | default, hover (underline) |
| Inline-edit input | fokus, valid, invalid (roed border) |
| Toggle | av (muted), paa (accent), loading (spinner i tracker) |
| "Lagre" inline | default, loading, success (groenn check fade) |
| Save-bar | hidden (form clean), slide-in (form dirty) |
| Toast | slide-in hoeyre, auto-dismiss 3 sek |
| Farezone | border-destructive/30 + bg-destructive/5 |

### Empty / loading / error / success

- **Loading:** Skeleton-form med 4 felt-rader
- **Error:** Per-felt roed tekst + retry-link
- **Success:** Groenn toast bunn-hoeyre "Endring lagret"
- **Tom seksjon:** F.eks. "Ingen API-noekler ennaa. Lag din foerste ->"

### Mobil-versjon

- Sub-nav blir horisontal scroll-tab eller dropdown oeverst
- Form-area tar full bredde
- Save-bar blir sticky bunn med full bredde-knapper
- Modaler blir bottom-sheet

## Pakke 1/5: CoachHQ Coach-profil

# AK Golf Platform -- CoachHQ -- Coach-profil

## Identitet

- **Produkt:** CoachHQ
- **URL:** `/admin/profil`
- **Arketype:** F -- Settings + profile
- **HTML-referanse:** `wireframe/screen-deck/coachhq/coach-profil.html`
- **Tilhoerende modaler:** `AvatarUploadModal`, `ChangePasswordModal`

## Spec -- hva skjermen er for

Coach-profilen er Anders sin egen "om meg"-side i CoachHQ. Profilbilde, navn, tittel, kort bio og spesialfelt (sertifiseringer, spraak, klubb-tilknytning) vises baade her og paa den offentlige siden `akgolf.no/coach/anders`. Endringer her propagerer dit.

## Layout -- UNIKT for denne skjermen

To-kolonne form-layout:

### Venstre kolonne (320px sticky)
- Stor avatar (160x160) med "Endre ->" -> `AvatarUploadModal`
- Navn (Geist 24px) + tittel under (muted 14px)
- Status-badge: "Aktiv coach" (accent-pill)
- 3 stat-kort vertikalt: "38 spillere", "12 aar som coach", "GFGK + WANG"
- Lenke: "Se offentlig profil ->" (aapner ny fane)

### Hoeyre kolonne (form-seksjoner)

**Personalia:** Fullt navn (Anders Kristiansen), Tittel (Hovedcoach AK Golf Academy), E-post, Mobil, Foedselsdato.

**Profesjonelt:** Bio (textarea), Sertifiseringer (chips: PGA of Norway, TPI Level 2, Mac O'Grady MORAD), Spraak (chips: Norsk, Engelsk), Klubb-tilknytning (GFGK, WANG), Coaching-stil (rich-text).

**Galleri:** 4 bilder i 2x2 grid med "+ Last opp"-plassholder.

## OEnsket output

1. Lyst tema, fullt utfylt profil
2. Moerkt tema
3. Inline-edit paa "Tittel" (input med fokus)
4. Hover paa galleri-bilde
5. Empty bio-state
6. Mobil <=640px

**CoachHQ-preferanser:** Default visning (Hub), Tett tabellvisning toggle, Auto-oppdater 30sek, Kompakt sidebar, Vis pyramide-fargekoder.

**Tema:** Lyst (default) / Moerkt / Foelg system med live preview.

**Farezone:** Eksporter alle data (GDPR), Slett konto.

## OEnsket output

1. Lyst tema, alle seksjoner default
2. Moerkt tema med live tema-preview
3. Inline-edit paa E-post med verifiserings-warning
4. Eksport pagaar (loading-state)
5. Farezone hover paa Slett konto
6. Mobil <=640px

- **Produkt:** CoachHQ

## Spec

API-skjermen er for utviklere og integrasjons-eiere som henter data ut av AK Golf HQ til andre systemer (TrackMan, GolfBox, Mailchimp).

## Layout

To hoved-faner: **API-noekler** | **Webhooks**.

**KPI-strip:** Aktive noekler (3 av oo Pro), API-kall siste 30d (14 280), Snitt-respons (84 ms), Feilrate (0,12%).

**Tabell:**
- akgolf-portal-prod | ak_pk_8f2... | read:plans, read:players | for 12 min siden | 14. mars 2026
- Mailchimp-sync | ak_pk_4a1... | read:players, read:emails | i gaar 22:14
- TrackMan-import | ak_pk_b3e... | write:sessions | for 3 timer siden

CTA: "+ Ny API-noekkel" -> `NewAPIKeyModal`.

### Fane Webhooks

**Tabell:**
- zapier-hooks (booking.created/cancelled) - Aktiv - for 8 min (200)
- akgolf-portal/webhooks/plans (plan.published/updated) - Aktiv - for 2t (200)
- test.akgolf.no/* (alle events) - **Feilet** - i gaar (502)

CTA: "+ Nytt webhook" -> `WebhookConfigModal`. Test-knapp per webhook.

**Dokumentasjon-kort:** "API-dokumentasjon ->", "Webhook-eksempler ->".

## OEnsket output

1. Lyst tema, fane API-noekler (3 noekler)

## Pakke 5/5: CoachHQ Tilgjengelighet (coach-kalender)

# AK Golf Platform -- CoachHQ -- Tilgjengelighet

## Identitet

- **Produkt:** CoachHQ
- **URL:** `/admin/tilgjengelighet`
- **Arketype:** F (kalender-konfig-variant)
- **HTML-referanse:** `wireframe/screen-deck/coachhq/tilgjengelighet.html`

## Spec

Tilgjengelighet styrer naar Anders kan bookes for 1:1 coaching. Det Anders setter her, vises i `booking.akgolf.no` for kunder. To hovedmoduser: **Standard uke** og **Unntak + ferier**.

## Layout

### Fane Standard uke

7 rader (Mandag-Soendag), hver med toggle "Tilgjengelig" + tids-slot-rader (Fra/Til + Pause). "+ Legg til slot" + "Kopier til andre dager ->" per dag.

Eksempel:
- Mandag: 09:00-12:00, 13:00-17:00 (paa)
- Onsdag: 09:00-12:00 (paa, kort dag)
- Loerdag: 10:00-14:00 (paa, helg-takst)
- Soendag: av

### Side-panel hoeyre (sticky)

**Booking-regler:** Min varsel (2t), Maks framover (60 dager), Buffer mellom timer (0 min), Tillat samme-dag (paa), Auto-godkjenn (av).

**Forhaandsvisning:** Mini-kalender + "AApne booking-side som kunde ->".

### Fane Unntak + ferier

Tabell med Periode | Type | Beskrivelse:
- 1.-7. juli 2026 | Ferie | Sommerferie
- 15. mai 2026 | Kurs | MORAD-oppdatering Stockholm
- 22. mai 14:00-17:00 | Blokkert | Privat turnering Bossum
- Hver onsdag 18:00-20:00 (recurring) | Familie | Junior-trening med datter

CTA: "+ Legg til unntak" -> `BlockTimeModal`. Sekundaer: "+ Recurring-regel".

**KPI-strip:** Tilgjengelig timer (38), Bookede timer (26 = 68%), Kommende ferier (2), Snitt-varsel (18t).

## OEnsket output

1. Lyst tema, fane Standard uke
2. Moerkt tema, fane Unntak (med recurring)
3. Side-panel utvidet med booking-regler
4. Kopier-til-andre-dager popover aapen
5. Konflikt-toast (en booking beroeres)
6. Mobil <=640px (segmented + stables)
