## Felles arketype-B spec (gjelder alle skjermer her)

# Batch 2 -- List + filter (Arketype B)

**Antall pakker:** 22 (18 skjermer + 4 tilhoerende modaler)
**Status:** Klar for claude.ai/design
**Estimert tid:** 4-6 timer

## Hvorfor denne batchen naa

Arketype B (List + filter) er det mest brukte moensteret i hele plattformen -- dominerer baade CoachHQ og PlayerHQ. Naar dette moensteret er stabilt, kan resterende lister i batch 6+ bruke samme grunnmal. Pilot-batch 1 (Dashboard) etablerte estetikken; denne batchen etablerer **interaksjonsmoenstre for tabell-data**.

## Arketype B -- felles spec (gjelder alle 18 skjermer)

Disse moenstrene skal vaere konsistente paa tvers av alle list-skjermer. Vis variasjon i innhold og kolonner, ikke i layout.

### Layout

```
+---------------------------------------------+
|  Sidebar  |  Hero (italic editorial title)  |
|           +---------------------------------+
|           |  KPI-strip (4 kort, valgfritt)  |
|           +---------------------------------+
|           |  Filter-bar:                    |
|           |  [Soek] [Chip 1] [Chip 2] [Sort] |
|           |                          [+ Ny] |
|           +---------------------------------+
|           |  Tabell / kort-grid             |
|           |  +--+---------+-----+----+---+  |
|           |  |  | navn    | ... |... | -> |  |
|           |  +--+---------+-----+----+---+  |
|           +---------------------------------+
|           |  Pagination eller "Last mer v"  |
|           +---------------------------------+
```

### Kolonner som ALLTID maa designes

| Element | States aa designe |
|---|---|
| Soek-felt | default, focus, with-text, clear-button, no-results |
| Filter-chip (multi-select) | default, hover, selected, disabled, count-badge |
| Sort-dropdown | default, open, item-hover, item-selected (med pil) |
| Sort-icon paa kolonne-header | default, hover, sort-asc, sort-desc |
| Rad | default, hover (subtil bg-shift), selected, klikk -> detail |
| Status-pill (per status) | unike farger per status -- bruk pyramide-paletten |
| "..."-aksjons-meny per rad | default, hover, popover-open |
| Bulk-checkbox | uvalgt, valgt, partial (hvis grupperinger) |
| "+ Ny X" CTA (primary) | default, hover, active, focus, disabled, loading |
| Pagination | default per side, hover, active page, disabled (forrige/neste) |

### Empty / loading / error / count-states

- **Empty (ingen data ennaa):** Stort sentrert omraade med ikon + "Ingen {ressurs} ennaa. Lag din foerste ->" CTA
- **Empty (filter gir 0):** "Ingen treff for filteret. Tilbakestill ->"
- **Loading:** 5 graa skeleton-rader
- **Error:** Per-tabell-error med retry
- **Count:** "Viser 12 av 38" eller "12 spillere" etter filter

### Mobil-versjon

- Filter-bar collapser til en "Filter (3) v"-knapp som aapner bottom-sheet
- Tabell konverterer til kort (en kort per rad, alle kolonner stables vertikalt)
- Bulk-actions skjules; "..."-meny per rad bevares

### Responsive breakpoints

- Desktop: >=1024px -- full tabell, alle filtre synlige
- Tablet: 768-1023px -- tabell med kondenserte kolonner, filtre delvis synlige
- Mobil: <=640px -- kort-layout

## Pakke 4/7: NewPlanModal

# AK Golf Platform -- Modal -- NewPlanModal

## Identitet

- **Produkt:** CoachHQ
- **Trigger:** `+ Ny plan`-CTA paa `/admin/plans` (CoachHQ Plans-list, pakke 01)
- **Type:** Multi-step wizard-modal (4 steg)
- **Tier-gating:** Ikke relevant (CoachHQ -- kun coaches)
- **HTML-referanse:** `wireframe/screen-deck/shared/modals/new-plan.html`
- **Audit:** `wireframe/audit/coachhq-plans.md` (modal-seksjon)

## Designsystem

Bruk **`branding-style-guide.html`** + **`design-system-v2.md`** som lastet system-kontekst. CoachHQ-modal -- bruker `card`-bg paa lyst tema, `popover`-bg paa moerkt. Maks 3 lime per modal. Eksakte token-navn -- ikke hardkode hex.

## Spec -- hva modalen er for

NewPlanModal lar coach (Anders K.) lage en ny treningsplan for en spiller paa under 2 minutter. Wizard guider gjennom: velg spiller + periode -> velg mal eller start blank -> tilpass oevelser -> bekreft og send. Alternativ AI-generering tilbys i steg 2. Etter steg 4 sendes plan til spiller for godkjenning og lukker modal.

## Layout -- UNIKT for modal

- **Modal-container:** Sentrert, max-width 720px, hoeyde auto, `rounded-xl` (12px), bakdrop `rgba(0,0,0,0.5)` med subtle blur(4px)
- **Header (sticky, 72px):**
  - Tittel italic Instrument Serif 24px: "Ny treningsplan"
  - Sub: steg-progress "Steg {n} av 4 - {steg-navn}"
  - Lukk-X oeverst-hoeyre (lucide `X` 20px)
- **Steg-indikator:** 4 prikker under header (accent for aktiv, primary for fullfoert, muted for ufullfoert) + linjer mellom
- **Body (variabel hoeyde, max 70vh, scrollbar):**
  - **Steg 1 -- Velg spiller + periode:**
    - Spiller-autocomplete med avatar-resultater (henter fra `/admin/elever`, viser Markus Pedersen som foerste eksempel)
    - Periode-velger: dato-fra + dato-til (default: "9. mai - 30. jun 2026")
    - Pyramide-fokus-velger: 5 slidere (FYS/TEK/SLAG/SPILL/TURN) som summeres til 100%
  - **Steg 2 -- Mal-valg:**
    - 3 store cards i grid: "Bruk mal ->" (aapner `TemplateSelectorModal`, pakke 22) - "AI-generer ->" (lime accent, hoeyre-stripe) - "Start blank ->"
  - **Steg 3 -- Tilpass oevelser:**
    - Liste over auto-foreslaatte oevelser (12 stk basert paa pyramide-fokus)
    - Per rad: drag-handle + oevelse-navn + kategori-pill + varighet + "x"-fjern
    - "+ Legg til oevelse"-link nederst (aapner soeke-popover)
  - **Steg 4 -- Bekreft + send:**
    - Sammendragskort: spiller, periode, antall oekter, pyramide-fordeling
    - Sjekkbokser: "Send til Markus for godkjenning" (default paa) - "Notifiser foresatt" (default av)
    - Footer-tekst: "Markus faar varsel og kan godta/avvise i sin portal"
- **Footer (sticky, 72px):**
  - Venstre: "Avbryt"-knapp (sekundaer)
  - Hoeyre: "<- Tilbake" + "Neste ->" (primary, accent-pill) -- i steg 4 endres "Neste" til "Send forslag ->"

## Klikkbare elementer

| Element | States |
|---|---|
| Steg-indikator-prikk | static, klikkbar tilbake til fullfoerte steg, ikke fremover |
| Spiller-autocomplete | default, focus, typing, suggesting, no-results, selected |
| Dato-felt | default, open (kalender), valgt-state |
| Pyramide-slider | default, dragging, summen-validering (100% paakrevd -- viser warning hvis ikke) |
| Mal-card | default, hover (lift), klikk -> laster valgt sti |
| OEvelse-rad | default, hover (drag-handle synlig), drag-active (rotert 2deg), drop-zone |
| "+ Legg til oevelse"-link | default, hover, focus, klikk -> soeke-popover |
| Sjekkboks | uvalgt, valgt, focus |
| "Avbryt"-knapp | default, hover, focus, klikk -> confirm-popover hvis endringer gjort |
| "<- Tilbake"-knapp | default, hover, focus, disabled (steg 1) |
| "Neste ->" / "Send forslag ->"-CTA | default, hover, active, focus, disabled (validering feil), loading (under AI-generering / send) |
| Lukk-X | default, hover, focus, klikk -> confirm-popover hvis endringer gjort |

## Empty / loading / error / validation-states

- **Steg 1 validering:** Pyramide != 100% -> warning-tekst under sliderne; periode tom -> felt roedt
