## Arketype G -- felles spec

Booking er forskjellig fra baade CoachHQ/PlayerHQ (logged-in app) og akgolf.no (ren marketing). Dette er en **transaksjons-wizard** som fungerer baade public (anonym kunde) og logged-in.

### Layout - public/logged-in hybrid

```
+-------------------------------------------+
|  Top-nav (alltid):                        |
|  AK Golf - Booking          Min side ->   |
+-------------------------------------------+
|  Booking-progress-stripe (steg 1-5):      |
|  [1.Tjeneste] -> [2.Tid] -> [3.Info]      |
|  -> [4.Betaling] -> [5.Bekreftelse]       |
+-------------------------------------------+
|                                           |
|  Hovedinnhold (max-width 600-1000px):     |
|  - Hero (italic editorial)                |
|  - Sub-info (mono caps)                   |
|  - Selve UI for steget                    |
|                                           |
+-------------------------------------------+
|  Footer-actions:                          |
|  [<- Tilbake]              [Fortsett ->]  |
+-------------------------------------------+
```

**INGEN app-sidebar.**

### Top-nav

- Venstre: "AK Golf - Booking" (Geist 14px, primary)
- Hoeyre: "Min side ->" (link, aapner login-popup hvis ikke innlogget)

### Booking-progress-stripe

5 pills med "->" mellom: 1.Tjeneste, 2.Tid, 3.Info, 4.Betaling, 5.Bekreftelse. Aktiv = primary fill. Ferdig = muted + check. Kommende = muted. Mobil collapser til "Steg X av 5" + 8px progress-bar.

### Wizard-flow

- Hvert steg har EN CTA fremover ("Fortsett ->")
- "<- Tilbake" som ghost
- Ingen sidesteg, ingen tabs
