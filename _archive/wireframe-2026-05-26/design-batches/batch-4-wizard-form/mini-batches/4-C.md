# Mini-batch 4-C - Modaler

**6 modaler i denne mini-batchen.** Felles moenster: Wizard / Form (Arketype D) levert som modal -- enten multi-step wizard eller single-step velger.

**Generer alle 6 modaler i denne sesjonen.** Foelg arketype-D-felles spec for fellestrekk + modal-spesifikke regler.

---

## Felles arketype-D + modal-spec

### Modal-spesifikke regler
- Max-width 720px (Payment 560, TemplateSelector 960, BookingConfirmation 560)
- `rounded-xl` (12px), bakdrop `rgba(0,0,0,0.5)` med blur(4px)
- Header sticky 72-96px med tittel italic + steg-progress + lukk-X
- Footer sticky 64-96px med standard knapp-struktur
- Lukk-X eller "Avbryt" -> confirm-popover hvis endringer er gjort
- Mobile: full-screen modal (`rounded-none`)

### Multi-step regler (gjelder alle modal-wizards)
- Dots ELLER numbers oeverst (ikke lineaer progressbar)
- Aktiv steg: accent-prikk. Fullfoert: primary. Ufullfoert: muted
- Klikkbar TILBAKE til fullfoerte steg, ikke fremover
- Footer: "Avbryt" venstre + "<- Tilbake" + "Neste ->" / handlings-CTA hoeyre
- "Tilbake" disabled paa steg 1
- Submit-CTA bytter tekst per steg ("Send forslag ->", "Bekreft og betal kr X ->", "Bruk valgt mal ->")

### Submit-states
idle -> validating -> submitting (CTA spinner + footer disabled) -> success (accent flash + redirect / auto-close) -> error (toast inni modal)

## Pakke 1/6: NewPlanModal

# AK Golf Platform -- Modal -- NewPlanModal

## Identitet

- **Trigger:** "+ Ny plan"-CTA paa /admin/plans
- **Type:** Multi-step wizard-modal (4 steg)
- **HTML-referanse:** `wireframe/screen-deck/shared/modals/new-plan.html`
- **Note:** Duplikat fra batch 2 -- inkludert som arketype-D-referanse for modal-wizards

## Spec

NewPlanModal lar coach (Anders K.) lage ny treningsplan for spiller paa under 2 minutter. Wizard guider gjennom: velg spiller + periode -> velg mal eller blank -> tilpass oevelser -> bekreft og send.

## Layout

Max-width 720px. Header 72px med "Ny treningsplan" italic + steg-progress + lukk-X. 4 dots steg-indikator.

### Steg 1 -- Spiller + periode
Spiller-autocomplete med avatar-resultater (Markus Pedersen som foerste eksempel). Dato-fra + dato-til. Pyramide-fokus-velger (5 slidere -> 100%).

### Steg 2 -- Mal-valg
3 store cards: "Bruk mal ->" / "AI-generer ->" (lime accent) / "Start blank ->".

### Steg 3 -- Tilpass oevelser
Liste 12 auto-foreslaatte oevelser. Drag-handle + navn + kategori-pill + varighet + X. "+ Legg til oevelse"-link.

### Steg 4 -- Bekreft + send
Sammendragskort + sjekkbokser ("Send til Markus" / "Notifiser foresatt"). Footer-CTA bytter til "Send forslag ->".

## OEnsket output

1. Steg 1 lyst tema (Markus valgt, periode satt, pyramide 100%)
2. Steg 2 lyst tema (3 mal-cards, hover paa "AI-generer")
3. Steg 3 lyst tema (12 oevelser, en under drag)
4. Steg 4 lyst tema (sammendrag + sjekkbokser)
5. Moerkt tema (steg 2)
6. Steg 2 AI-generering loading-state
7. Validation steg 1 (pyramide 95%, warning under)
8. Mobile full-screen (steg 1)
9. Avbryt-confirm-popover
