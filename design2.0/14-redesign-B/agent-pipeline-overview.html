Slik fungerer det

1. Signal kommer fra database-trigger, webhook eller cron. Hvert signal har severity (info/warning/urgent/critical) og payload.
2. Skill abonnerer på relevante signal-typer. En skill er ren funksjon — analyserer signal + henter kontekst, returnerer beriket data.
3. Agent mottar skill-output og bestemmer plan-aksjon. Aksjoner kan være auto-godkjent (lav risk) eller havne i godkjenningskø.
4. Coach godkjenner/avviser i CoachHQ → aksjon utføres → resultat logges som nytt signal.
